# Multi-Host Ping Monitor Daemon

Легковесный и надежный Python-скрипт для одновременного мониторинга нескольких серверов (IP-адресов или доменных имен) с мгновенным уведомлением в Telegram при их оживании. Предназначен для развертывания на Linux VPS в качестве системного демона (`systemd`).

---

## ✨ Особенности
* **Поддержка миграций (DNS)**: Можно указывать как IP, так и домены. Скрипт нативно использует системный утилиту `ping`, поэтому при смене IP у домена он автоматически подтянет новые данные из DNS.
* **Динамический список**: Опрашивает все хосты параллельно в цикле. Как только один из серверов оживает, бот присылает уведомление, убирает его из списка и продолжает следить за оставшимися.
* **Zero Spam**: Не присылает уведомления, пока серверы лежат. Вы получите сообщение только при запуске и в момент поднятия хоста.
* **Production-ready**: Настройки изолированы в `.env` (12-Factor App), критические ошибки логируются, а управление полностью делегировано `systemd` (автозапуск, логирование в journald).

## 🛠 Установка и развертывание

### 1. Клонирование и подготовка окружения
Перейдите в директорию с вашими ботами и создайте папку проекта:
```bash
cd /root/bots/ping_monitor_bot
```

Создайте изолированное виртуальное окружение и установите зависимости:
```bash
python3 -m venv venv
./venv/bin/pip install -r requirements.txt
```

### 2. Настройка конфигурации
Создайте файл .env в корне проекта:
```bash
nano .env
```

```ini
Добавьте ваши настройки (хосты перечисляются через запятую):
TARGET_HOSTS=example.com, 192.168.1.10, my-vps-server.ru
TG_TOKEN=123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ
CHAT_ID=987654321
INTERVAL=600
PING_COUNT=10
```

⚠️ Важно: Перед запуском убедитесь, что вы запустили вашего Telegram-бота (нажали /start), иначе API вернет ошибку 403 Forbidden.

### 3. 🚀 Настройка системного демона (systemd)
Чтобы скрипт работал в фоне 24/7 и автоматически поднимался при перезагрузке VPS, используйте готовый конфигурационный файл сервиса из репозитория.
#### 1. Откройте файл ping-monitor.service и актуализируйте пути **WorkingDirectory** и **ExecStart** под вашу систему:
```bash
nano ping-monitor.service
```
Убедитесь, что абсолютные пути указаны корректно (символ ~ использовать нельзя):
```ini
WorkingDirectory=/root/bots/ping_monitor_bot
ExecStart=/root/bots/ping_monitor_bot/venv/bin/python3 ping_monitor.py
```
#### 2. Скопируйте настроенный файл сервиса в системную директорию systemd:
```bash
cp ping-monitor.service /etc/systemd/system/
```
#### 3. Перезагрузите конфигурацию systemd и включите автозапуск демона:
```bash
systemctl daemon-reload
systemctl enable --now ping-monitor
```

## 📊 Управление и логирование

Проверить статус службы:
```bash
systemctl status ping-monitor
```

Посмотреть логи в реальном времени:
```bash
journalctl -u ping-monitor -f
```

Перезапустить мониторинг (например, после изменения .env):
```bash
systemctl restart ping-monitor
```

Остановить службу вручную:
```bash
systemctl stop ping-monitor
```

### После того как все указанные хосты успешно поднимутся, скрипт отправит финальное уведомление в Telegram и корректно завершит свою работу, перейдя в статус inactive (dead), чтобы не утилизировать ресурсы VPS.
